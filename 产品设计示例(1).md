# 职业蓝图产品设计

## 一、产品定位

职业蓝图不是职业百科，也不是招聘平台，而是面向学生和职场新人的 AI 时代职业认知与路径探索工具。

第一版只解决三个问题：

1. 我第一次能不能看懂这个职业真实在做什么。
2. 这个职业和其他职业如何连接、可转向哪里。
3. AI 会怎样改变它，我现在下一步该做什么。

## 二、页面定版

| 页面       | 优先级 | 定版结论                           | 首版验收                   |
| ---------- | ------ | ---------------------------------- | -------------------------- |
| 首页       | P0     | 职业星图 + 热门职业 + 蓝图生成入口 | 用户能点击职业节点进入详情 |
| 职业详情页 | P0     | 8 张卡片固定结构                   | 12 个岗位均可完整展示      |
| 蓝图生成页 | P0     | 3 个问题，不做严肃测评             | 用户 1 分钟内能完成        |
| 蓝图结果页 | P0     | Top 3 推荐 + 推荐理由 + 下一步行动 | 推荐结果可解释             |
| 职业对比页 | P1     | 5 组固定对比                       | 用户能看懂两个职业差异     |
| 分享卡片   | P1     | 蓝图结果和职业对比可分享           | 可生成静态分享内容         |

## 三、首页

### 页面目标

让用户快速理解这是一个“看懂职业连接和 AI 影响”的产品，并进入职业探索或蓝图生成。

### 首屏信息

| 模块   | 内容                                 |
| ------ | ------------------------------------ |
| 主标题 | AI 时代，你的职业路线会被重写吗？    |
| 副文案 | 先看懂一个职业，再决定下一步怎么走。 |
| 主按钮 | 生成我的职业蓝图                     |
| 次按钮 | 先看看热门职业                       |

### 职业星图规则

| 元素     | 规则                                         |
| -------- | -------------------------------------------- |
| 节点     | 12 个 P0 岗位全部展示                        |
| 节点颜色 | 按 AI 影响等级区分：新生、重构、增强、低影响 |
| 节点大小 | 首版固定，不做复杂热度算法                   |
| 连线     | 展示核心职业关系，优先展示强度 4-5 的关系    |
| 点击     | 点击节点进入职业详情页                       |

## 四、职业详情页

### 页面目标

把一个职业讲清楚，不能只堆 JD，也不能写成百科长文。

### 8 张卡片顺序

| 顺序 | 卡片         | 字数建议  | 重点                                             |
| ---- | ------------ | --------- | ------------------------------------------------ |
| 1    | 一句话定义   | 50 字以内 | 学生能听懂                                       |
| 2    | 真实工作任务 | 3-5 条    | 来自 JD 或访谈                                   |
| 3    | 一天怎么过   | 4 个区块  | 上午、下午、协作对象、交付物                     |
| 4    | 能力要求     | 4 类      | 硬技能、软技能、工具、加分项                     |
| 5    | 入门路径     | 5 项      | 入门周期、学习内容、入门项目、作品集、第一周行动 |
| 6    | AI 影响      | 5 项      | 替代、增强、保留、新机会、一句话结论             |
| 7    | 职业连接     | 4 类      | 相似、协作、可转岗、进阶                         |
| 8    | 下一步行动   | 4 个按钮  | 收藏、对比、生成蓝图、学习路径                   |

### 内容底线

1. 每个岗位不少于 20 条资料支撑。
2. 任何结论必须来自资料、访谈或明确人工判断。
3. 不允许把 AI 生成内容直接当原始事实。
4. 文风短句化，避免行业黑话。

## 五、蓝图生成

### 页面目标

用 3 个问题给用户一个探索型推荐，不制造“测评决定人生”的错觉。

### 3 个问题

| 问题                       | 选项                                                                       | 影响维度 |
| -------------------------- | -------------------------------------------------------------------------- | -------- |
| 你更愿意面对哪类工作？     | 数据分析 / 产品设计 / 内容表达 / 技术实现 / 沟通合作 / 视觉创作            | 岗位族群 |
| 你更喜欢哪种工作方式？     | 独立研究 / 团队协作 / 快速执行 / 长期打磨 / 对外沟通                       | 工作风格 |
| 你对 AI 的态度更接近哪种？ | 想用 AI 提效 / 想做 AI 产品 / 想用 AI 创作 / 想理解底层技术 / 暂时只想避坑 | AI 适配  |

### 匹配规则

1. 每个选项命中一组岗位标签。
2. 每个岗位按命中标签累积分。
3. 同分时优先推荐学生友好、入门路径清晰、内容完整度高的岗位。
4. 结果只输出 Top 3，不输出低分岗位。

### 风险提示文案

```text
这不是职业测评结论，只是基于你当前选择生成的探索建议。真正适合你的方向，还需要结合项目实践、作品和真实反馈继续验证。
```

## 六、蓝图结果页

| 模块       | 内容                                        |
| ---------- | ------------------------------------------- |
| 结果标题   | 你的 AI 时代职业探索方向                    |
| Top 3 职业 | 岗位名称、匹配理由、AI 影响等级、下一步行动 |
| 高亮星图   | 高亮推荐岗位和相邻连接                      |
| 下一步     | 看详情、加入对比、生成学习路径、分享到群    |

## 七、职业对比页

### 首批 5 组

1. 数据分析师 vs 商业分析师。
2. 产品经理 vs 新媒体运营。
3. 前端开发 vs 后端开发。
4. AI 产品经理 vs 产品经理。
5. 内容创作者 vs 新媒体运营。

### 对比维度

| 维度       | 展示重点                     |
| ---------- | ---------------------------- |
| 核心工作   | 每天主要做什么               |
| 入门难度   | 学习门槛和作品门槛           |
| AI 影响    | 被替代、被增强、新机会       |
| 常用工具   | 首批常见工具，不做全量工具库 |
| 适合人群   | 哪类学生更容易上手           |
| 下一步建议 | 如果纠结，先做什么验证       |

## 八、分享卡片

| 类型         | 内容                                 |
| ------------ | ------------------------------------ |
| 蓝图结果分享 | 用户 Top 3 职业、关键词、二维码/链接 |
| 职业对比分享 | 两个职业的核心差异和下一步建议       |
| 职业详情分享 | 一句话定义、AI 影响、一条行动建议    |

首版可以先生成静态分享卡片，不强制接入复杂海报生成能力。

## 九、验收清单

- [ ] 首页能进入 12 个职业详情。
- [ ] 职业详情页 8 张卡片顺序一致。
- [ ] 蓝图生成 3 个问题能生成 Top 3。
- [ ] 推荐理由能解释到岗位标签。
- [ ] 职业对比 5 组可查看。
- [ ] 分享内容不夸大、不暗示测评结论。
- [ ] 移动端阅读不拥挤，按钮不遮挡正文。
